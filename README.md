# Reflection

## How can I ensure that my code, program, or software is functional and secure?

I ensure that software is functional and secure by following requirements carefully and validating those requirements through comprehensive testing. In this course, I used JUnit tests to verify expected functionality, validate input constraints, and test boundary conditions. By testing both valid and invalid scenarios, I was able to identify defects early and improve software reliability. Security is further strengthened through input validation and by ensuring data integrity throughout the application.

## How do I interpret user needs and incorporate them into a program?

I begin by analyzing the project requirements and identifying what the user expects the software to accomplish. These requirements are translated into classes, methods, and test cases that verify the intended behavior. Throughout this course, I learned that creating tests directly from requirements helps ensure that user needs are accurately implemented and that the software performs as expected.

## How do I approach designing software?

I approach software design by first reviewing requirements and breaking the system into smaller, manageable components. I focus on creating clear class structures, maintaining separation of responsibilities, and designing software that is easy to test and maintain. This course reinforced the importance of incorporating testing into the design process early, which improves code quality, simplifies debugging, and results in more reliable software.
